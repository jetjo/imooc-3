1、全局安装rollup
            npm install --global rollup

        2、直接执行rollup仅打印help信息

        3、ES6没有导出default，则以下面的导入报错
            import user1 from './user'
            应改为：import * as user1 from './user'

            
                📌以import * as obj from ''的模式导入模块时，千万不要对obj的属性进行写操作，除非这个属性是一个setter 

                比如：

                模块m如下： 

                

                    const num = 99;
                    import {num};
                

                引用模块的模块如下： 

                

                    import * as obj form './m'
                    obj.num++//这句会报错，因为num在m模块中是个常量！！！
                

                ★总结：ES6模块就像一个单例的类，对于其成员不要进行写，要想改变成员的值，因该通过模块导出的方法
            

        4、如果打包到一起时，模块的变量与引用者的变量处于同一作用域且重名，模块的变量会被改名
            
        5、总之，访问未定义变量会报错
        
        6、ES模块就像镜子，无论导出多上次，导出的这些都是一个实例，导出就像照镜子，照多少次，镜子中都是一个人
